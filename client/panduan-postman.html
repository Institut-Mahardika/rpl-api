00 — Persiapan

Setup Postman Web

ℹ️ Kenapa Postman Web, bukan aplikasi desktop?
Postman Web bisa dipakai langsung di browser tanpa install apapun. Cocok untuk lab kampus yang tidak mengizinkan instalasi software. Semua fitur yang kita butuhkan tersedia di versi web.
1

Buka Postman Web

Buka browser, kunjungi web.postman.co. Klik Sign Up jika belum punya akun, atau Sign In jika sudah. Akun gratis sudah cukup.

💡 Gunakan email kampus atau Gmail
Daftar dengan Google Account lebih cepat — klik "Continue with Google".
2

Buat Workspace baru (opsional tapi disarankan)

Di sidebar kiri klik Workspaces → Create Workspace. Beri nama misalnya "Praktikum API Institut Mahardika". Workspace membantu memisahkan request praktikum dari yang lain.

3

Buat Collection baru

Di sidebar klik ikon Collections (ikon folder) → klik tombol + → pilih Blank Collection. Beri nama "API Mahasiswa". Collection adalah tempat menyimpan semua request supaya tidak hilang.

✓

Siapkan Base URL

Minta IP dan port server ke dosen di kelas. Catat base URL-nya:

Base URL server dosen
http://192.168.1.10:3000 ↑ IP dosen ↑ port
⚠️ Pastikan satu jaringan WiFi
Laptop kamu dan laptop dosen harus terhubung ke WiFi yang sama. Berbeda jaringan = request tidak sampai.
01 — Request pertama

GET — Ambil Semua Data

GET

Lihat semua mahasiswa yang sudah terdaftar

/api/mahasiswa
1

Buat request baru di collection

Klik kanan collection "API Mahasiswa" → Add request. Beri nama "GET Semua Mahasiswa".

2

Atur method dan URL

Pastikan dropdown method menunjukkan GET. Ketik URL berikut di kolom URL:

GET Semua Mahasiswa
GET
http://192.168.1.10:3000/api/mahasiswa
ℹ️ GET tidak butuh Body
Request GET hanya butuh URL. Tidak perlu buka tab Body atau tambahkan Header apapun.
3

Klik Send dan lihat response

Scroll ke bawah — response muncul di panel bawah. Kalau berhasil, status menunjukkan 200 OK.

Response
200 OK 48 ms · 312 B
{ "status": "success", "message": "2 data mahasiswa ditemukan", "total": 2, "data": [ { "npm": "624C0004", "nama": "Budi Santoso", "prodi": "Teknik Informatika", "kelas": "TI-6A", "pesan": "Halo!", "status": "active" } ] }
✓

Simpan request

Klik Save (Ctrl+S / Cmd+S). Request tersimpan di collection dan bisa dipakai lagi kapan saja.

02 — Kirim data

POST — Daftarkan Data Diri

POST

Tambah data mahasiswa baru ke server

/api/mahasiswa
⚠️ POST berbeda dengan GET
Request POST mengirim data ke server. Kamu perlu mengatur tiga hal: method, header Content-Type, dan body JSON.
1

Buat request baru, pilih method POST

Tambah request baru di collection. Klik dropdown method (sebelah kiri kolom URL) → pilih POST. Isi URL:

POST Daftar Mahasiswa
POST
http://192.168.1.10:3000/api/mahasiswa
2

Atur Body — tab Body → raw → JSON

Klik tab Body di bawah kolom URL → pilih radio button raw → di dropdown sebelah kanan pilih JSON (bukan Text). Postman otomatis menambahkan header Content-Type.

Tab Body
Params Headers Body Pre-req
● none   ○ form-data   ○ x-www-form   ● raw   ○ binary     JSON ▼
{ "npm": "624C0004", "nama": "Nama Kamu", "prodi": "Teknik Informatika", "kelas": "TI-6A", "pesan": "Halo dari Postman!" }
💡 Ganti dengan data dirimu
Isi npm dengan NPM aslimu (contoh: 624C0004), nama dengan nama lengkap, dan seterusnya. Field pesan boleh dikosongkan atau dihapus.
3

Klik Send — response harus 201 Created

Response
201 Created 62 ms · 198 B
{ "status": "success", "message": "Data berhasil ditambahkan", "data": { "npm": "624C0004", "nama": "Nama Kamu", "prodi": "Teknik Informatika", "kelas": "TI-6A", "pesan": "Halo dari Postman!", "status": "active" } }
⚠️ Kalau muncul 409 Conflict
NPM kamu sudah terdaftar sebelumnya. Cek dengan GET all apakah datamu sudah ada. Satu NPM hanya boleh didaftarkan satu kali.
03 — Path parameter

GET — Cari by NPM

GET

Ambil data satu mahasiswa berdasarkan NPM

/api/mahasiswa/:npm

Endpoint ini menggunakan path parameter — NPM dimasukkan langsung ke dalam URL. Tidak ada body, tidak ada header tambahan.

1

Isi URL dengan NPM di bagian path

GET Mahasiswa by NPM
GET
http://192.168.1.10:3000/api/mahasiswa/624C0004
ℹ️ Path parameter vs Query string
Path parameter: /api/mahasiswa/624C0004 — NPM jadi bagian URL.
Query string: /api/mahasiswa?npm=624C0004 — ini berbeda dan tidak akan bekerja di endpoint ini.
✓

Response 200 — data satu mahasiswa

Response
200 OK35 ms
{ "status": "success", "message": "Data ditemukan", "data": { "npm": "624C0004", "nama": "Nama Kamu", ... } }

Coba juga NPM yang tidak ada → harusnya muncul 404 Not Found.

04 — Update data

PUT — Update Data

PUT

Ubah sebagian atau seluruh data mahasiswa

/api/mahasiswa/:npm
1

Method PUT + URL dengan path parameter

PUT Update Mahasiswa
PUT
http://192.168.1.10:3000/api/mahasiswa/624C0004
2

Body → raw → JSON — isi field yang ingin diubah saja

Tidak perlu kirim semua field. Cukup field yang mau diubah:

Body — hanya field yang berubah
Body
● raw   JSON ▼
{ "pesan": "Pesan saya sudah diupdate via Postman" } // cukup field yang berubah saja // field lain (npm, nama, dll) tidak perlu dikirim
✓ Response 200 — data setelah diupdate
Server akan membalas dengan data terbaru. Cek field yang kamu ubah sudah berubah di response.
05 — Request hapus

DELETE — Request Hapus Data

DELETE

Kirim permintaan hapus — masuk antrian dosen

/api/mahasiswa/:npm
⚠️ Data tidak langsung terhapus
Request DELETE akan membuat statusmu masuk ke antrian pending. Dosen yang akan approve atau reject dari panel admin mereka. Jika di-approve, data dihapus permanen.
1

Method DELETE + URL dengan path parameter

DELETE tidak butuh Body — hanya URL dan method yang benar.

DELETE Request Hapus
DELETE
http://192.168.1.10:3000/api/mahasiswa/624C0004
✓

Response — status pending

Response
200 OK41 ms
{ "status": "pending", "message": "Permintaan hapus untuk NPM 624C0004 sudah dikirim. Menunggu persetujuan dosen.", "data": { "npm": "624C0004", "nama": "Nama Kamu" } }
06 — Produktivitas

Tips Postman yang Perlu Tahu

Environment Variable — supaya tidak perlu ketik URL berulang

Daripada ketik http://192.168.1.10:3000 di setiap request, simpan sebagai variable:

1

Buat Environment

Klik ikon Environments di sidebar → + → beri nama "Server Dosen".

2

Tambah variable BASE_URL

Environment: Server Dosen
VARIABLEINITIAL VALUE BASE_URLhttp://192.168.1.10:3000
✓

Gunakan di URL setiap request

GET
{{BASE_URL}}/api/mahasiswa

Kalau IP dosen berubah, cukup update di satu tempat — semua request ikut berubah.

Tabel error yang sering muncul di Postman

Status Artinya Yang harus dicek
200 OK Request berhasil, data ditemukan —
201 Created Data baru berhasil dibuat —
400 Bad Request Data yang dikirim tidak lengkap atau formatnya salah Pastikan semua field wajib terisi dan JSON valid (tidak ada koma berlebih)
404 Not Found NPM tidak ditemukan atau URL salah Cek penulisan URL — pastikan ada /api/mahasiswa/ sebelum NPM
409 Conflict NPM sudah terdaftar atau sudah di antrian hapus Gunakan NPM yang belum pernah didaftarkan
422 Unprocessable Format nilai field salah (terlalu pendek, terlalu panjang) NPM maks 10 karakter, Nama min 2 karakter
Could not get response Server tidak bisa dihubungi Cek IP, Port, dan koneksi WiFi — pastikan satu jaringan dengan dosen
✓ Checklist sebelum selesai praktik
  • GET semua data — muncul list mahasiswa dari server
  • POST data diri — response 201, dataku muncul di GET all
  • GET by NPM — response 200 dengan data spesifik milikku
  • PUT update — ubah pesan, response 200, perubahan terlihat di GET
  • DELETE request — response pending, dosen konfirmasi menerima request
