00 — Sebelum mulai

Persiapan

1
Buka file template
File yang dibagikan dosen: client-mahasiswa-template.html
⚠️ Jangan buka file langsung dari folder
File HTML yang melakukan fetch ke server lain harus dibuka via web server lokal, bukan double-click. Jika dibuka langsung, browser akan memblokir request karena alasan keamanan (file:// origin).
2
Buka via VS Code Live Server
Cara paling mudah tanpa install apapun

Install extension Live Server di VS Code (oleh Ritwick Dey), lalu klik kanan file HTML → Open with Live Server. File akan terbuka di browser dengan URL http://127.0.0.1:5500/... — ini sudah benar.

💡 Alternatif jika tidak ada VS Code
Buka terminal di folder file, jalankan: python3 -m http.server 5500, lalu buka http://localhost:5500 di browser.
3
Buka DevTools — tab Console
Wajib terbuka selama praktik untuk melihat error

Tekan F12 (Windows) atau Cmd+Option+I (Mac) → pilih tab Console. Jika ada error JavaScript, akan muncul di sini dengan warna merah beserta nomor barisnya.

01 — Langkah pertama

Koneksi ke Server Dosen

Di bagian atas halaman ada dua input: IP dan Port. Isi dengan IP dan port yang diberikan dosen di kelas.

contoh isian
IP   : 192.168.1.10
Port : 3000
↓ preview otomatis terbentuk
URL  : http://192.168.1.10:3000

Klik tombol Cek Koneksi. Kalau berhasil, indicator di pojok kanan atas akan berubah hijau dan muncul toast "✓ Server terhubung".

✓ IP dan Port tersimpan otomatis
Tidak perlu isi ulang setiap reload halaman — sudah disimpan ke localStorage browser.
⚠️ Kalau gagal terhubung
Pastikan laptop kamu dan laptop dosen berada di jaringan WiFi yang sama. Jika beda jaringan, request tidak akan sampai ke server.
02 — Tugas 1

Fungsi ambilSemua()

GET
Ambil semua data mahasiswa dari server
Endpoint: GET /api/mahasiswa

Fungsi ini dipanggil saat tombol ↓ Ambil Data diklik. Tugasmu ada dua bagian: isi URL yang benar, lalu render datanya ke tabel.

Bagian 1 — Isi URL endpoint

ambilSemua() — baris URL
sebelumsesudah 
const url = BASE_URL + '/api/...'   // ← GANTI INI
const url = BASE_URL + '/api/mahasiswa'

Bagian 2 — Render data ke tabel

Setelah tbody.innerHTML = '', tambahkan kode berikut untuk mengisi tabel:

ambilSemua() — render tabel
// Tambahkan di bawah: tbody.innerHTML = ''

if (json.data.length === 0) {
  tbody.innerHTML = '<tr><td colspan="6" style="text-align:center;padding:32px;color:#7A8694">Belum ada data</td></tr>'
  return
}

json.data.forEach(m => {
  const stClass = m.status === 'active' ? 'st-active' : 'st-pending'
  const row = `
    <tr>
      <td class="td-npm">${m.npm}</td>
      <td>${m.nama}</td>
      <td>${m.prodi}</td>
      <td>${m.kelas}</td>
      <td class="td-pesan" title="${m.pesan || ''}">${m.pesan || '—'}</td>
      <td><span class="td-status ${stClass}">${m.status}</span></td>
    </tr>
  `
  tbody.innerHTML += row
})
🔍 Cara cek hasilnya
Klik ↓ Ambil Data. Kalau server masih kosong, tabel akan menampilkan "Belum ada data". Kalau ada data, muncul dalam baris-baris tabel. Toast "✓ N data berhasil dimuat" harus muncul di pojok kanan bawah.
03 — Tugas 2

Fungsi submitDaftar()

POST
Kirim data diri ke server
Endpoint: POST /api/mahasiswa

Fungsi ini dipanggil saat tombol Kirim Data diklik. Ada 3 baris yang harus diisi: URL endpoint, method, dan Content-Type header.

submitDaftar() — 3 baris yang diganti
sebelumsesudah 
const url = BASE_URL + '/api/...'   // ← GANTI INI
const url = BASE_URL + '/api/mahasiswa'

const res = await fetch(url, {
  method:  '...',                    // ← GANTI INI
  method:  'POST',
  headers: { 'Content-Type': '...' } // ← GANTI INI
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ npm, nama, prodi, kelas, pesan }) // sudah ada
})
ℹ️ Kenapa harus ada Content-Type?
Header Content-Type: application/json memberitahu server bahwa data yang kamu kirim adalah JSON. Tanpa ini, server tidak tahu cara membaca body request dan akan merespons error 422.
🔍 Cara cek hasilnya
Isi form NPM, Nama, Prodi, Kelas, lalu klik Kirim Data. Response box di bawah tombol akan menampilkan HTTP 201 dan data yang tersimpan. Tabel semua data otomatis ter-refresh. Kalau NPM yang sama dikirim dua kali → muncul HTTP 409 (konflik).
04 — Tugas 3

Fungsi cariByNpm()

GET
Cari satu mahasiswa berdasarkan NPM
Endpoint: GET /api/mahasiswa/:npm

Fungsi ini menggunakan path parameter — NPM dimasukkan langsung ke dalam URL, bukan di body.

cariByNpm() — susun URL dengan path parameter
sebelumsesudah 
const url = BASE_URL + '/api/...' + npm  // ← GANTI INI
const url = BASE_URL + '/api/mahasiswa/' + npm

// contoh hasil URL jika npm = '624C0004':
// http://192.168.1.10:3000/api/mahasiswa/624C0004
⚠️ Jangan lupa slash sebelum npm
'/api/mahasiswa/' — ada garis miring di akhir sebelum ditambah npm. Kalau tidak ada, URL jadi /api/mahasiswa624C0004 yang tidak valid.
🔍 Cara cek hasilnya
Isi NPM yang sudah terdaftar (misal NPM kamu sendiri setelah POST), klik Cari. Response box menampilkan HTTP 200 dan satu data mahasiswa. Coba NPM yang tidak ada → muncul HTTP 404.
05 — Tugas 4

Fungsi updateData()

PUT
Update data mahasiswa yang sudah ada
Endpoint: PUT /api/mahasiswa/:npm

Sama seperti GET One, URL menggunakan path parameter. Bedanya kali ini ada method: 'PUT' dan body berisi field yang ingin diubah.

updateData() — 3 baris yang diganti
sebelumsesudah 
const url = BASE_URL + '/api/...' + npm  // ← GANTI INI
const url = BASE_URL + '/api/mahasiswa/' + npm

const res = await fetch(url, {
  method:  '...',                    // ← GANTI INI
  method:  'PUT',
  headers: { 'Content-Type': '...' } // ← GANTI INI
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(body) // sudah disiapkan otomatis
})
✓ Tidak perlu isi semua field
Kode sudah otomatis hanya mengirim field yang kamu isi. Kalau hanya mau ubah pesan, cukup isi NPM dan field Pesan Baru — field lain dikosongkan.
🔍 Cara cek hasilnya
Isi NPM datamu, ubah salah satu field (misal ubah Pesan), klik Update Data. Response box menampilkan HTTP 200 dan data terbaru. Tabel otomatis refresh — cek kolom yang kamu ubah sudah berubah.
06 — Tugas 5

Fungsi requestHapus()

DEL
Kirim permintaan hapus data
Endpoint: DELETE /api/mahasiswa/:npm

Method DELETE tidak membutuhkan body. Cukup URL dengan path parameter dan method yang benar. Data tidak langsung terhapus — masuk ke antrian pending dan menunggu persetujuan dosen.

requestHapus() — 2 baris yang diganti
sebelumsesudah 
const url = BASE_URL + '/api/...' + npm  // ← GANTI INI
const url = BASE_URL + '/api/mahasiswa/' + npm

const res = await fetch(url, {
  method: '...'   // ← GANTI INI
  method: 'DELETE'
})
// tidak ada body — DELETE hanya butuh URL dan method
🔍 Cara cek hasilnya
Isi NPM datamu, klik Request Hapus. Response box menampilkan status pending dan pesan "menunggu persetujuan dosen". Toast kuning muncul. Data di tabel masih ada dengan status berubah jadi pending delete. Dosen yang akan approve atau reject dari panel admin mereka.
07 — Kalau ada masalah

Troubleshoot Error

Error / Gejala Penyebab Solusi
CORS policy blocked File dibuka via file:// bukan server lokal Buka via VS Code Live Server atau python3 -m http.server
Failed to fetch / ERR_FAILED IP atau Port salah, atau beda jaringan WiFi Minta IP terbaru dari dosen, pastikan WiFi sama
HTTP 404 URL endpoint salah atau NPM tidak ditemukan Cek penulisan /api/mahasiswa/ — pastikan ada slash sebelum NPM
HTTP 409 NPM sudah terdaftar atau sudah pending hapus Gunakan NPM yang belum pernah didaftarkan
HTTP 422 Format data salah — tipe atau panjang field tidak sesuai Cek NPM maks 10 karakter, Nama min 2 karakter
HTTP 500 Error di sisi server dosen Hubungi dosen, bukan salah kamu
Tabel tidak tampil meski ambil data berhasil Kode render tabel belum ditambahkan Pastikan kode json.data.forEach(...) sudah ada di fungsi ambilSemua()
Toast tidak muncul saat klik tombol Ada syntax error di JavaScript Buka Console (F12), cari baris merah, perbaiki typo di sana
💡 Langkah debug universal
Setiap kali ada error yang tidak jelas: 1) Buka Console → baca pesan error → cari nomor barisnya. 2) Buka tab Network di DevTools → klik request yang gagal → lihat tab Response untuk melihat pesan error dari server.
